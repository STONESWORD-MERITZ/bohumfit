# BOHUMFIT 실비(실손) 청구 안내 기능 — 설계 문서 v3
> 작성일: 2026-06-06
> 성격: 신규 기능 설계 (구현 기준 문서). Cowork/Codex 작업의 기준.
> 핵심 원칙: **추정 안내형** — 확정 금액이 아니라 "청구 가능성/추정 범위"를 안내.
> v2 대비 변경: SURIT-022 A 데이터 진단 결과 반영 +
>   급여/비급여 처리 방식 확정(PDF 금액=급여 자동집계 / 비급여=사용자 직접 입력).

## A. 데이터 진단 결과 (SURIT-022, 현 파이프라인 기준)
- **급여/비급여 구분**: 현재 불가에 가까움. PDF 원시 레코드에 "내가 낸 의료비"
  컬럼은 담기나, disease_aggregator 는 총진료비만 집계하고 "내가 낸 의료비" 미집계.
  → **확정 처리(아래)**: PDF "내가 낸 의료비"는 전부 **급여 본인부담금으로 간주**해
    연도별 집계. 비급여는 PDF에 의존하지 않고 **사용자 직접 입력**으로 분리.
- **요양병원 구분**: 요양기관명만 있고 요양병원 플래그 없음 → 신규 모듈에서
  요양기관명 휴리스틱 + 연간 합산 필요.
- **연도별 합산**: 모든 레코드가 진료개시일(YYYY-MM-DD) 보유 → 연도 추출 가능,
  단 비용 연도별 합산 로직은 신규 구현 필요.
---
## 1. 기능 개요
알릴의무(고지) 분석에 쓴 **동일한 PDF 데이터**를 재사용하여, 사용자가 가입한
실손보험 기준 **청구 가능성**과 **상한제 환급 가능성**을 안내한다.
### 3대 안내 (한 화면에 통합)
1. **실손 청구 가능성** — 기본진료 의료비 기준, 세대별 자기부담률 적용
2. **실손 자기부담금 상한제** — 연 자기부담금이 세대별 상한 초과 시 환급 가능성
3. **건강보험 본인부담상한제** — 연 급여 본인부담금이 소득분위별 상한 초과 시 환급 가능성
### 절대 원칙
- **확정 금액 단정 금지.** "추정", "OO~OO원 수준일 수 있음", "청구 대상일 수 있음"으로만.
- 모든 결과에 "정확한 금액·보장 여부는 보험사·공단 확인 필요" 명시.
- 보험 모집·중개·상품 추천·가입 권유 금지 (표시규제 의식).
- 알릴의무 결과의 "확인/기록" 톤과 동일.

## 2. UI 구조 (확정)
### 별도 탭 추가
- 현재 ResultView 는 `productTab: "standard" | "easy"` 탭 구조 보유 (Disclosure.tsx:550)
- 여기에 **세 번째 탭 "실손 청구"** 추가 (예: `productTab: "standard" | "easy" | "insurance"`)
- 또는 별도 영역으로 분리 — 구현 시 UI 흐름에 맞춰 결정
- **재업로드 불필요**: 분석 결과(`result`)와 기본진료 데이터를 그대로 재사용 (코드 확인됨)
- 단, 비저장 원칙상 페이지 새로고침 시 재업로드 필요 (알릴의무와 동일, 정상)
### 입력 영역 (실손 탭 내)
- 실손 세대 선택 (1~5세대 + 모름)
- 세대 내 옵션 (3세대 비급여 20/30% 등 — 선택)
- 비급여 직접 입력 (고액 진료 금액 — 선택)
- 소득분위 선택 (건보 상한제용, 1~10분위 구간 + 모름)

## 3. 계산 로직 (공제 제외 → 동일 입력=동일 출력)
### 3-1. 핵심 단순화 (사용자 확정)
- **통원 공제금액(회당 1~2만원)은 적용하지 않음.**
- 세대별 자기부담률만 적용 → 같은 데이터 + 같은 세대면 항상 같은 결과.
- "추정" 단어로 정확성 한계 표현.
### 3-1-b. 급여/비급여 처리 방식 (확정)
- **급여 본인부담금** = PDF "내가 낸 의료비" 컬럼을 연도별로 집계 (전부 급여로 간주).
- **비급여** = PDF에 의존하지 않고 **사용자가 직접 금액 입력** (없으면 0).
- 즉 PDF 금액에는 비급여를 섞지 않고, 비급여는 사용자 입력으로만 반영.
  → 건보 상한제(급여만 대상)와 실손 비급여 계산이 깔끔히 분리됨.
### 3-2. ① 실손 청구 가능성
```
[급여분] 각 연도/행의 PDF "내가 낸 의료비"(=급여 본인부담):
  청구 추정액 = 급여 본인부담 × (1 - 세대별 급여 자기부담률)
[비급여분] 사용자 직접 입력 금액:
  청구 추정액 = 비급여 입력액 × (1 - 세대별 비급여 자기부담률)
출력: "청구 대상일 수 있음" + 추정 범위 (급여/비급여 합산)
```
### 3-3. ② 실손 자기부담금 상한제
```
연간 실손 자기부담금 누적 = Σ(급여 자기부담분 + 비급여 자기부담분)
세대별 자기부담금 연 상한 초과 시 → "초과분 추가 보장 가능성" 안내
```
### 3-4. ③ 건강보험 본인부담상한제
```
연간 급여 본인부담금 누적 = Σ(PDF "내가 낸 의료비")  ※진료일 기준 연도별, 급여만
  ※ 비급여 입력분은 건보 상한제 대상 아님 (제외)
소득분위별 상한액(2026 표) 초과 시 → "공단 환급 가능성" 안내
요양병원 입원 120일 초과 시 별도 상한 적용 (요양기관명 휴리스틱으로 분기 시도,
  불가하면 "요양병원 장기입원 시 상한 다름" 안내만)
```

## 4. 수치 테이블
### 4-1. ⚠️ 세대별 실손 자기부담률 (검색 초안 — 약관 검증 필요)
> 출처: 보험사 블로그·뉴스. **코드 상수로 박기 전 세대별 표준약관으로 검증 필수.**

| 세대 | 기간 | 급여 자기부담 | 비급여 자기부담 | 비고 |
|---|---|---|---|---|
| 1세대 | ~2009.9 | 거의 0% (입원), 통원 소액공제 | 〃 | 손보 0%/생보 20%, 상품별 편차 큼 |
| 2세대 | 2009.10~2017.3 | 10~20% | 20% | 표준화 시작 |
| 3세대 | 2017.4~2021.6 | 10~20% | 20% 또는 30% (선택 옵션) | 비급여 3종 특약 분리 |
| 4세대 | 2021.7~2026.5 | 20% | 30% | 비급여 할증제 |
| 5세대 | 2026.5.6~ | 입원 20% / 외래 건보연동(최대 60%) | 비중증 50% | 중증 비급여 상한 신설 |

※ 1세대·3세대는 상품별 편차가 커서 세대 내 옵션 필요.
※ 5세대 급여 외래 건보연동·중증/비중증 분리는 최신이라 정밀 검증 필요.
### 4-2. ⚠️ 실손 자기부담금 연 상한 (확정 필요)
> 세대별로 다름 (예: 일부 세대 연 200만원 등). 약관 확인 후 확정.
### 4-3. ✅ 건보 본인부담상한제 2026 (공식 확정 — 공단 재난상한제운영부-82, 2026.1.6.)

## 5. 데이터 처리 (SURIT-022 A 진단 반영 — 확정)
1. **급여 본인부담금** — PDF "내가 낸 의료비" 컬럼을 연도별 집계 (전부 급여 간주).
   현재 disease_aggregator 가 이 컬럼을 집계하지 않으므로, 실손 모듈에서
   원시 레코드 기준으로 "내가 낸 의료비"를 연도별 재집계하는 로직 신규 추가.
2. **비급여** — 사용자 직접 입력 (PDF 미사용). 없으면 0.
3. **요양병원 120일 초과** — 요양기관명 휴리스틱으로 분기 시도. 불가 시 안내만.
4. **연도별 누적** — 진료개시일(YYYY-MM-DD)로 연도 추출, 연 단위 비용 재집계.

## 6. 구현 가이드 (Cowork 작업 시)
- **수치는 전부 상수 파일/모듈로 분리** — 세대별 자기부담률, 실손 상한, 건보 상한제 표.
  매년/약관 변경 시 한 곳만 수정. 각 상수에 기준연도·출처 주석.
- **계산 모듈은 알릴의무 로직과 독립** — 실손 계산이 고지 판정에 영향 없음.
- **데이터 재사용** — 기존 `result`/기본진료 데이터 공유, 재업로드 없음.
- **개인정보 비저장** — 세대·소득분위·비급여 입력 모두 저장 안 함 (세션 내만).
- **출력 톤** — "추정", "~일 수 있음", "보험사·공단 확인 필요" 일관 적용.
---
## 7. 면책
본 기능은 청구·환급 "가능성"을 안내하는 보조 도구이며, 실제 보험금·환급금 지급 여부와
금액은 보험사 약관·심사 및 국민건강보험공단 기준에 따라 결정됩니다. 세대별·연도별 수치는
변경될 수 있으며 정확한 계산은 약관·공단 확인이 필요합니다. 본 도구는 보험 모집·중개·권유를
목적으로 하지 않습니다.
