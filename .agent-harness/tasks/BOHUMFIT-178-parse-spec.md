### 2.2 p6~7 — 상품별 가입현황 (소스 B: 회사×담보 매트릭스, 핵심)
- 회사(계약)가 **열**, 담보가 **행**. 열이 4개 초과면 **페이지 분할**(p6=계약 1~4, p7=계약 5~6). 좌측 요약열은 **각 페이지 반복**.
- 열 헤더 5행: `(n) 회사명` / `상품명(wrap)` / `{납입기간}납/{만기} 만기` / `{계약시작}~{계약만기}`(종신=`9999.12.31`) / `{월보험료}원`.
- 좌측 요약 헤더: `6` / `573,227`(계약 수·월납합계).
- 매트릭스 행: `[대분류라벨(병합,좌측)] [담보명] [요약값] [계약1] … [계약N]`.
- **좌측 요약값 = 담보별 합산**(정액). 미가입 셀 = `-`.

금액 토크나이저(중요):
```
"N억 N,NNN만"  → N×100,000,000 + NNNN×10,000     (예: "5억 5,000만"=550,000,000)
"N억"          → N×100,000,000                    (예: "2억"=200,000,000)
"N,NNN만"/"N만" → ×10,000                          (예: "1,000만"=10,000,000, "27만"=270,000)
"-"            → 미가입(0 / null)
```
열 배정은 헤더 `(n)`의 x좌표로 클러스터(pdfplumber word x0) 권장 — `-layout` 공백분할은 폭 넓은 금액에서 열 밀림 위험.

## 5. Step 4 — 엣지케이스·리스크

1. **회사명 중복**(삼성화재 idx4·5, KB손보 idx1·2): 집계/매트릭스 키를 **계약 idx**로. 표 헤더는 `(idx) 회사명 + 상품명`으로 구분 표기.
2. **미가입 셀 `-`**: 값 0, 담보 전체 미가입이면 `enrolled:false`·상태 `미가입`(p20).
3. **납입기간 이상치 `81년`(972월)**: 규칙대로 그대로(전기납 100세). 단 `paid_total`이 79.9백만으로 급증 → 표에 "전기납" 주석 권장(캡 없음, task="그대로").
4. **종신/9999**: 만기=종신 표기, 총납입은 **납입기간(유한)** 사용(예: idx6 15년납).
5. **rep 담보 대표값 기준**(실손 다건 시): 문건주는 실손이 idx4 단건 → 대표=그 값. 다건 시 기준 필요 → **권고: 최신 계약일 우선, 동률 시 최대금액**. §7-5.
6. **KB 외 타사 제안서**: `Producer=oz` + 헤더 문구로 KB 표준 판별. 불일치 시 파싱 거부·안내("KB 신정원 표준 양식 전용"). 범위 밖(후속).
7. **금액 토큰 변형**: "억"만/"만"만/콤마/공백/`-` 모두 처리(§2.2 토크나이저).
8. **좌표 밀림**: 폭 넓은 금액("5억 5,000만")에서 `-layout` 공백분할 오배정 위험 → pdfplumber word x0 클러스터 권장.

## 7. ★Human 결정 필요 (Open Decisions)

1. **입원일당 집계 (합산 vs 대표값)** — 태스크 규칙=대표값이나 KB PDF는 **합산**(6만)으로 표기. 정액 일당은 회사별 중복 지급(실제 6만/일 수령)이라 **합산이 현실적**. → *권고: 합산*. rep 유지 시 KB 요약열을 무시하고 대표값 재계산 필요. **확정 요망.**
2. **11 대분류 미포함 담보 4종**(장기요양간병비·경증치매진단·간병일당2). → *권고: 대분류 12 "간병/치매" 신설*(또는 입원/기타 편입). **확정 요망.**
3. **일상생활배상(가족/일상/자녀배상)**: 비례보상형 → *권고: 대표값(rep)*. 문건주 자료는 idx4 단건(1억)이라 sum=rep(차이 없음)이나, 다건 가입 시 sum이 과대계상되므로 rep 권고. **확정 요망.**
4. **수술비(암/뇌/심장수술비) 분류**: KB는 "수술" 묶음. 11 대분류에 암/뇌/심장 별도 존재 → *권고: "수술" 대분류로 묶되 소분류(장기) 태그 유지*.
5. **실손 다건 시 대표값 기준**: *권고: 최신 계약일 우선(실손 최근 가입 유효), 동률 시 최대금액*.

> 위 결정은 상품·보상 정책 성격 → Cowork 자동 확정하지 않고 flag. 179 착수 전 확정 권장.
