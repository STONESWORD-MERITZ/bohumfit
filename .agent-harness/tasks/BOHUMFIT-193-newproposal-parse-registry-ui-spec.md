### S0-2. ⚠ 불일치·확인 플래그 (패스2/약관/Human)

1. **통합치료비 bundle 분해(암수술비1,750·표적8,050·방사선2,050·뇌혈관수술2,000·심장수술2,000):** 가입금액(cap)은 실측되나(또또암 68=5천·78=8천·82=1억·172=5천·174=3백; 종합 151=1억), **치료유형별 지급액(subBenefit)은 약관/상품설명서 sub-rate 필요.** 급여/비급여 합산 범위에 따라 값이 달라짐 → **Human②·③.**
2. **또또암 암진단 대표값:** 기본계약 `32 암종별(30종)통합암진단비` 세부보장은 암종별 **4천9백만원(4,900)** 표기인데 골든은 **5,000**. 대표 가입금액 규칙(4,900→5,000 반올림/헤드라인 캡) 확인 필요.
3. **미래 유사암:** 미래 M-케어에 유사암진단특약(2,000만원 추정)이 보이나 골든 유사암(1,010=종합10+또또암1,000)에 **미포함.** 후 선택에서 제외인지, 다른 행 귀속인지 확인.
4. **KB 심장 잉여 담보:** KB `125 심장질환(특정Ⅱ)` 5천만원·`132 부정맥(Ⅰ49)` 1백만원은 골든 미반영(급성심근경색·부정맥row 공란/대체). 매핑 규칙(특정Ⅰ→부정맥row, 특정Ⅱ→급성심근경색은 공란) 확정.
5. **가족일상배상 중복 귀속:** 종합 616(1억)이 골든값. KB에도 가족일상배상 존재 → 회사간 합산 대상이 아니라 **대표 1개(종합)**로 처리한 것으로 보임 — 배상책임 집계규칙(합산 vs 대표) 확인.
6. **자부상 500:** 운전자 자동차사고부상치료비 tier(1급3,000~14급30, 5급=450)에서 500 도출 근거 불명 → **Human④.**

## S1. 회사별 파서 어댑터 규칙

공통 인터페이스: `adapter(pdf_bytes) -> [{담보번호?, 담보명, 가입금액_만원, 보험료?, 납기?, 만기?, 갱신형?, parent?, tier?, raw}]`. 회사 감지 후 어댑터 분기.

### S1-1. 헤더/회사 탐지
- 상품명 문자열로 회사·상품 판정(예 "메리츠 또 걸려도", "미래에셋 M-케어", "KB 금쪽같은", "메리츠 The좋은알파Plus종합", "메리츠 운전자상해"). 실손(현대해상)은 유지계약 경로.

### S1-2. 계약·담보 행경계
- 메리츠/운전자: `^\s*\d+\s+갱신형?\s*<담보명>\s*<금액>\s*<보험료>` 패턴. 담보번호 앵커. `기본계약`·`선택계약` 섹션.
- KB: `^\d+\s*\[?(기본계약|의무부가특약)?\]?...` + 갱신형. 담보명/금액 줄분리 재결합(다음 줄의 `<금액> <보험료> (갱신종료:...)` 흡수).
- 미래: `[<라벨>] <금액>` 블록 스캔.

### S1-3. 부모-자식(세부보장 참조·`┗`) — 179b 재사용
- `┗` 또는 상위 담보번호 아래 들여쓰기 자식(`67/68 ┗ 암진단및치료비[...]`) = 부모의 subBenefit. 179b의 detail/parse_detail_pages 접근을 재사용.
- "세부보장 참조"(암종별통합암진단 → 별도 세부표) = 별도 페이지 집계. 대표 가입금액 규칙 적용.

### S1-4. ★한글 금액 파싱 (→ 만원 정수)
실측 표기 사전: `1천만원=1,000` · `4천9백만원=4,900` · `1억원=10,000` · `2억원=20,000` · `2백40만원=240` · `4백50만원=450` · `5천만원=5,000` · `8천만원=8,000` · `3백만원=300` · `10만원=10` · `20만원=20` · `30만원=30` · `5백만원=500` · `1천2백만원=1,200` · `9백만원=900` · `1백만원=100` · `5만원=5`.
- 규칙: 억→10,000, 천만→1,000, 백만→100, 십만→10, 만원 단위 정수. 복합("4천9백만"=4,900) 누적 파싱. 파싱 실패는 **드롭 금지 → ParseWarning**.

### S1-5. ★수술비Ⅱ 2열 분리
- 메리츠 `수술비Ⅱ(1-5종)[상해n종]`/`[질병n종]` 대괄호 태그로 **[상해]/[질병] 2열** 분리. 미래 통합형(1특약, 질병·재해 동일) = 양열 동일값.
- ★열좌표 기반 분리는 실 PDF 패스2 보정점(현재는 대괄호 텍스트 태그로 충분히 분리됨 — 실측 확인).

### S1-6. 보험료·납기·만기·갱신형·실패행
- `보험료(원)`·`납기/만기`(20년/20년, 20년/100세, 10년/10년) 캡처. `갱신형`·`(갱신종료:100세)` 플래그. 미인식 행 = **ParseWarning(드롭 금지)**, 설계사 확인 유도.

## S2. 레지스트리 스키마 + 집계규칙 + 시드

### S2-1. 스키마
```jsonc
{
  "insurer": "메리츠",                 // 회사
  "productKeys": ["또걸려도또받는암보험"], // 상품 매칭 키
  "riderKey": "암 통합치료비(기본형)",   // 담보 식별(담보번호/명)
  "type": "bundle | tier | simple",
  "category": "암/암수술비",            // 13분류·하위행 매핑
  "repRule": "sum | representative",   // 회사내부 중복 처리
  "subBenefits": [ { "label": "암수술비", "target": "암/암수술비", "amount": 0 } ]
}
```
- type 판정: **bundle**(cap 1개 + 약관상 치료별 상이지급 → 분해) / **tier**(부모 밑 자식 각 금액 — 수술비Ⅱ 1-5종) / **simple**(담보=하위행 직결).
- **미등록 특약 = 기타 + 경고** → 설계사 확인 후 **JSON 한 줄 추가로 성장(데이터=JSON, 코드 미변경).**

### S2-2. ★집계규칙 (골든에서 도출)
1. **회사 내부 중복(동일보장 상향판: 진단비 vs 진단비Ⅱ, 허혈성 vs Ⅱ) = 대표값 1개.**
   근거: 종합 뇌혈관진단비Ⅰ(1,000)+Ⅱ(1,000) → 골든 뇌혈관 종합기여 **1,000**(합산 아님). 허혈성 Ⅰ+Ⅱ → 1,000.
2. **회사 간 = 합산.**
   근거: 뇌혈관 4,000 = 종합1,000 + KB3,000. 상해사망 12,100 = 10,000+2,000+100.
3. **종수술(tier) = [질병|상해] 2열, 회사간 합산. 통합형 rider(미래 1특약)는 양열 동일.**
   근거: 1~5종 = 종합 + 미래 각 열 합산(위 S0-1).
4. **통합치료비(bundle) = cap 미표기. subBenefits로 분해 → 각 하위행(암수술비/표적/방사선/뇌혈관수술/심장수술)에 매핑 후 회사간 합산.**
   ⚠ sub 지급액은 약관 sub-rate(급여/비급여 합산범위) → Human②③.

### S2-3. ★시드 — 또또암 통합치료비 분해 worked example
골든 목표: 암수술비 **1,750** · 표적항암약물치료 **8,050** · 항암방사선약물치료 **2,050**.
실측 bundle 담보(또또암, cap):

| 담보 | 명칭 | cap(만원) |
|---|---|---|
| 68 | 암진단및치료비[암 통합치료비Ⅲ(비급여)] | 5,000 |
| 78 | 암 통합치료비(기본형)(암중점치료기관) | 8,000 |
| 82 | 암 통합치료비Ⅱ(비급여, 암중점치료기관) | 10,000 |
| 172 | 항암중입자방사선치료비 | 5,000 |
| 174 | 계속받는 항암방사선약물치료비(급여,연1회) | 300 |

- **분해 매핑(설계):** 각 통합치료비는 약관상 **수술/표적항암약물/방사선** 등 치료유형별 지급액이 정해짐. subBenefits[label→target, amount]로 분해해 하위행별 합산.
  - 표적항암약물치료 8,050 ≈ `78`(8,000) 계열 + α(50) → **정확 재현 위해 약관 sub-rate 필요(Human②).**
  - 항암방사선약물치료 2,050 ≈ `172`/`174` 계열 분해.
  - 암수술비 1,750 ≈ bundle 수술 subBenefit 합.
- ★이 3값의 **정확 재현은 약관 sub금액 확정 후** worked example 완결. 현 단계는 **담보번호·cap·분해 골격**까지 확정(실측), sub금액은 pass-2.
- **뇌·심장 수술비(각 2,000):** 종합 `151 특정순환계질환 통합치료비`(1억) bundle → 뇌혈관수술·심장수술 subBenefit 분해(Human③ 귀속 규칙).

### S2-4. 시드 배정(골든 전 담보 type/category/repRule)
- simple: 상해수술비·질병수술비·골절진단비·깁스치료비·가족일상배상·상해사망·상해후유장해·운전자 5종.
- tier: 수술비Ⅱ(1-5종)[상해|질병](종합), 1-5종수술(미래 통합).
- representative(회사내부): 뇌혈관진단비Ⅰ/Ⅱ(종합), 허혈성Ⅰ/Ⅱ(종합), KB 심장 특정Ⅰ/Ⅱ 매핑.
- bundle: 또또암 68/78/82/172/174, 종합 151.
- 미래 통합수술 → 2열 동일. KB 심장 특정Ⅰ→부정맥row, 허혈성→허혈성row(⚠특정Ⅱ 공란 규칙).
