# BOHUMFIT-124 알릴의무 분석 로직 버그 수정
## 배경
실제 심평원 PDF 2종 테스트 결과 2개 파싱 버그 발견.
## 버그 1: 한방 입원 미집계
- 현상: 자동차사고 한방병원 침구과 입원(주상병 BS134)이 입원일수로 집계 안 됨.
- 예상 원인: filters.py/disease_aggregator.py에서 B코드(한방 상병)를 비질병 제외하거나
  침구과/한방과를 입원 집계에서 누락.
- 확인: pipeline/disease_aggregator.py, filters.py, pipeline/helpers.py — B상병 처리·입원/외래 구분.
## 버그 2: 처방조제 PDF 투약일수 합산 미파싱
- 현상: 비뇨기과 동일 성분(methylprednisolone 등) 7일×반복 → 합산 30일+ 인데 미반영.
- 원인 후보: a) 처방조제 PDF 주상병코드 없어 기본진료정보 파서 실패
  b) _pharma_seen 키가 같은날짜+병원+약품을 외래/약국 중 하나만 카운트
  c) 투약일수가 진료일 합산 아닌 단일 MAX로만 처리.
- 확인: pipeline/pdf_parser.py, pipeline/disease_aggregator.py, filters.py(_pharma_seen,_max_presc).
## 작업 순서
- Step1 분석(수정 전): 처방조제/기본 구분 로직·B/Z/U 상병 필터·투약일수 집계 추적 → handoff "분석 결과" 기록.
- Step2 버그1: B상병/한방과 입원 누락 시 입원일수는 집계되도록(질병분류는 양방 기준 유지, 입원/외래 컬럼값 기준). 회귀 테스트.
- Step3 버그2: 처방조제 PDF 파싱 보정 + _pharma_seen 키에 날짜 포함(날짜별 중복제거+합산), 외래/약국 동일날짜·동일성분 중복제거. 회귀 테스트.
- Step4 검증: pytest 전체 + 신규 2케이스(B코드 한방 입원일수, 동일성분 반복 합산).
## 수정 금지 범위
- 태스크 명시 외 파일 수정 금지. 프런트엔드 수정 금지.
## 완료 조건
- pytest 전체 통과, 신규 케이스 통과, handoff에 변경/분석/검증 기록.
- ★실 PDF·PII 미커밋. 테스트는 합성 픽스처.
