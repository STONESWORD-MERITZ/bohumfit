## Owner
Cowork(자료조사+구현, 샌드박스) → Codex(Windows 검증·커밋·푸시). 기능·라우팅 변경 없음(/why 유지).

## 목표
랜딩 "왜 중요한가"(`/why`)를 **메커니즘 서사 + 검증 통계 카드 + 출처 각주 + CTA**로 재구성한다.
메커니즘 3축: ① 병력 → 청약서 서면 성실고지 ② 청구 시 보험사 서면 확인·구두 안내는 기록 없으면 인정 어려움 ③ 가입하는 순간보다 가입 후(청구 시점)가 중요.

## 자료조사 결과(요약 — 상세는 handoff)
- WebSearch(2026-06)로 고지의무 분쟁 **구체 비율 통계**는 1차 출처(금감원/소비자원) 수치로 **교차검증 불가** → 왜곡 방지 위해 **새 통계 수치 미채택**.
- 채택: 048에서 출처와 함께 정리된 **검증 가능 카테고리 통계만 유지**(설계사 수·대면 채널 비중). 분쟁 빈발은 **정성 서술**(금감원 소비자 유의 반복 안내)로만, 수치 단정 금지.
- 메커니즘 3축은 통계가 아니라 **고지의무 법·실무 구조**(서면 질문 대상·청구 시 서면 확인) → 수치 없이 사실 서술.

## 구현
- `whyContent.ts`에 `MECHANISM_STEPS`(3축, 수치 없음) 추가.
- `WhyDisclosure.tsx`에 메커니즘 섹션 추가(기존 검증 통계/분쟁 장면/CTA/면책 유지). 기존 톤/토큰(ink·accent·line)/폰트 유지, 반응형(grid), 접근성, `break-keep`.

## 금지/원칙
