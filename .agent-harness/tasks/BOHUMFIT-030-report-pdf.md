# BOHUMFIT-030 백엔드 리포트 PDF 출력 (고지/실손 분리)

## Owner
Cowork(Claude) — 구현 1차. 검증·커밋·푸시는 Codex(Windows).

## 목적
고지의무 점검 결과 / 실손 예상 보험금 분석을 각각 독립된 PDF 리포트로
백엔드에서 생성·다운로드한다. 디자인은 승인된 템플릿(네이비+골드, 40~50대 가독성)을 따른다.

## 범위 (In)
- 백엔드 PDF 생성 엔드포인트: 리포트 종류 파라미터 `report_type ∈ {disclosure, insurance}` (각각 별도 출력)
- 렌더 방식: HTML/CSS → 헤드리스 Chromium(예: playwright) PDF. 한글은 Noto CJK 서버 임베딩.
- 금액·판정은 분석 결과를 그대로 사용 — 산식 재구현/재계산 금지(insuranceCalc 미러·필터 결과 재사용)
- 사업자 정보 푸터(상호 보험핏 / 대표 이민규 / 사업자등록번호 174-29-01975 / 소재지 / 분석도구 BOHUMFIT)
- 건강정보 미저장: 생성 후 서버에 PDF·진료데이터 영구 저장 금지(휘발 처리)

## 콘텐츠 수정 사항 (이번 반영 6건)
1. [고지] '고지 불요 판단 기준'을 1~4 번호 줄바꿈(각 항목 한 줄)으로 표시
2. [고지] '확인 필요' 판정은 **추가검사/재검사(1년 질문) 해당 여부 미확정** 경우에만 사용.
   - 치료일수 경계 등 다른 사유는 결정론 결과(권고/불요)로 확정. '확인 필요'로 쓰지 말 것.
   - 나머지 판정 로직(권고/불요)은 현행 결정론 룰 그대로 — 회귀 없이 유지
3. [공통] 산출물 내 **'구 브랜드명' 문자열 전부 제거 → 'BOHUMFIT'** (문서번호 접두 SR-→BF-, 푸터 분석도구, 본문 문구 포함)
4. [실손/①] 비급여 항목은 "약관 확인"이 아니라 **"설계사가 수기 입력한 비급여 금액 기준 산출"** 임을 명시(안내 문구 포함)
5. [실손/본인부담상한제] 환급은 **급여 본인부담금 기준(비급여 제외)** 으로 산정·표기
6. [실손/자기부담금 상한제] 연 200만원 상한 규칙을 세대별로 명시
   - 1~3세대: **급여+비급여** 자기부담금 합계 200만원 초과 시 보험사 환급
   - 4세대 이후: **급여** 자기부담금 200만원 초과 시 보험사 환급

## 범위 밖 (Out)
- 코드베이스/태스크ID 접두사(구 브랜드명-) 일괄 리네임 → **별도 태스크(BOHUMFIT-031)**
- 결정론 룰 엔진(filters)·계산기 산식 변경 — 표현/표기만 수정, 계산 결과 불변
- 프런트 트리거 UI(다운로드 버튼) 연결은 v1 후속

- 대형 기존파일 편집 최소화. 마운트 git 금지(status 포함) — git은 Codex(Windows)
- Chromium/폰트 의존성은 `requirements.txt`/배포 설정에 추가(버전 고정)

## 검증
- 백엔드: `cd backend && python -m pytest -q` (리포트 생성 단위 테스트 신설: 종류별 PDF 바이트 생성·한글 깨짐 없음)
- 두 종류 PDF 실제 생성 후 육안 확인: 6건 수정 반영 / 구 브랜드명 미존재(grep) / 사업자 푸터 / 면책
- 금액 동일성: 리포트 수치 == 분석 결과(insuranceCalc/필터) — 재계산 없음 확인
- 프런트 변경 동반 시: tsc(app/node), lint, build

## 결정 필요 (코드 확인 후 handoff 기록)
- PDF 렌더러: playwright(Chromium) vs 가벼운 대안 — Railway 배포 용량/빌드 영향 확인 후 택1
- '확인 필요(추가검사/재검사)' 신호를 현재 분석 결과에서 어떻게 식별하는지(전용 플래그 유무)
- 엔드포인트 경로명/응답 형식(스트림 다운로드 vs base64)
