# BOHUMFIT-205 입원 회차별 표기 + 결과 조회기간(10~0년) + 약변경 3개월 2차확인

Owner flow: Claude Chat -> Claude Cowork -> Codex
Current owner: Codex

## Intent

사용자(설계사) 실사용 피드백 3건.

1. 간편고지 복사문·카드의 입원 항목이 `최초 진료일자 ~ 최종 진료일자 / 입원N일` 한 줄로 병합 표기되어,
   여러 번 입원한 경우(예: 전십자인대파열 3회 입원) 하나의 장기 입원처럼 읽히고 입원 일수도 혼동됨.
   → 입원 회차별(개시일~종료일/일수)로 각각 확인 가능해야 한다.
2. 고지 결과를 확인할 때 10년부터 0년까지 조회기간을 선택할 수 있어야 한다(기본 10년).
   → 간편 Q2만이 아니라 건강체·간편심사 결과 전체에 클라이언트 확인용 필터를 적용한다.
   실제 고지 대상은 상품별 청약서 문항 기간과 인수 기준으로 그대로 판단하며, 선택값은 법적 문항·서버 판정·복사문·PDF를 바꾸지 않는다.
3. 비식별 실자료 사례: 이전 처방과 동일한 약인데 "3개월이내 처방 변경" 고지대상으로 나옴.
   → 실데이터 재현: 동일 성분(모사프리드시트르산염수화물)의 제네릭 브랜드 전환(디스펩틴정→모사프리엠정),
     동일 성분·동일 용량 브랜드 전환(거드액→알지에액, 티로파정→스파론정)이 "새 약 추가"로 오탐.
     또한 중첩 괄호·PDF 줄바꿈 공백으로 정규화 키가 갈라지는 결함('가스티렌정)' 등 잔재 키) 확인.
   → 고지 후보가 생기면 3개월 이전 처방과 가장 최근 확인 처방을 성분·용량 기준으로 재대조한다.
      현재 처방이 이전과 완전히 같으면 제외하되, 새 성분 또는 실질 용량 증가는 유지한다.
