# BOHUMFIT-129 보험사 링크 페이지 정보 확장
## 배경
/insurance-links 페이지(전산·약관·팩스 중심)에 실무 정보 확장 + 상세보기 영역 신설.
## 스펙
1. 탭: 전체/손해/생명 → 전체/손해/생명/**공제회사**.
2. 데이터 확장 필드(없으면 null/""): customerCenter·incallNumber·helpdeskNumber·claimFormUrl·browser(Edge/Chrome/무관)·lastVerifiedDate·claimFaxSub·displayOrder(메리츠화재=1)·category(손해/생명/공제회사).
3. 카드 버튼: [전산][약관][청구양식][팩스복사] — 청구양식 URL 없으면 비활성(회색). 하단 우측 최종확인일(연회색), 권장 브라우저 배지(있을 때).
4. 상세보기 토글: 업무 연락처(고객센터·인콜·헬프데스크, 각 복사·없으면 "확인 필요") / 청구 정보(대표팩스·보조팩스(있을 때)·청구안내=faxNote) / 사용 환경(권장 브라우저).
5. 메리츠화재 상단 고정 — 동일 탭 내 displayOrder 오름차순.
6. 상세 하단 [고객 안내문 복사]: "[보험금 청구 안내]\n보험사: {명}\n청구팩스: {대표팩스}\n보험금 청구 전, 보험사별 필요서류와 청구 가능 기준은 해당 보험사 공식 안내를 함께 확인해 주세요."
## 작업: Step1 분석 → Step2 데이터 확장(+공제 더미 1~2) → Step3 UI → Step4 검증.
## 수정 금지: 고지의무 분석 파일·타 페이지·기존 보험사 값(전산/약관/팩스) 변경 금지(필드 추가만).
## 완료: tsc/build pass, 탭4, 청구양식 버튼, 상세보기, 메리츠 상단, handoff 기록.
