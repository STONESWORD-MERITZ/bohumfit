## A. 데이터 진단 결과 (현 파이프라인 기준)
- **급여/비급여 구분 = 부분(현 불가에 가까움)**. `pdf_parser.parse_single_pdf` 는 표 헤더→값을
  일반 캡처해 원시 레코드에 "내가 낸 의료비" 컬럼이 (있으면) 담기나, `disease_aggregator.py:265`
  는 `총진료비/진료비/본인부담총액/급여비용총액`만 비용으로 집계하고 "내가 낸 의료비" 미집계.
  심평원 '건강보험 요양급여내역'은 급여 기준이라 "내가 낸 의료비"=급여 본인부담 가능성 높으나
  PDF/문서 없이 비급여 혼재 단정 불가. → 설계 v3 확정: PDF 금액 전부 급여 간주 + 비급여는 사용자 입력.
- **요양병원 입원일수 = 부분**. 입원일수(`_inpatient_days_map`)는 질병별 집계되나 요양병원
  구분 플래그 없음(요양기관명만 캡처). 요양기관명 휴리스틱 + 연간 합산 신규 필요.
- **연도별 합산 = 데이터 가능/로직 미구현**. 전 레코드 `진료개시일`(YYYY-MM-DD) 보유로 연도 추출
  가능하나, 비용 연도별 합산 로직 없음 → 실손 모듈에서 원시 레코드 연 단위 재집계 신규.

## B. 수치 상수 (constants.py — 설계 v3 §4)
- 세대별 자기부담률(1~5, 급여/비급여, 3세대 비급여 20/30 옵션) — §4-1 (⚠️ 검색 초안, 약관 검증 필요)
- 실손 자기부담금 연 상한(세대별) — §4-2 (⚠️ 미확보 → None 자리만)
- 건보 본인부담상한제 2026(분위별 일반/요양병원120일초과, 사전급여 843만) — §4-3 (✅ 공식 확정)
- 각 상수에 기준연도·출처 주석. 매년/약관 변경 시 이 파일만 수정.

## C. 계산 모듈 (calculator.py — 설계 v3 §3, 순수 함수)
- 급여 = PDF "내가 낸 의료비" 연도별 재집계(전부 급여 간주). 비급여 = 인자 입력(없으면 0).
- ① 실손 청구 추정(급여/비급여 각 자기부담률) ② 실손 자기부담금 상한 초과 ③ 건보 상한제 초과(급여만, 분위 입력, 요양병원120일 휴리스틱)
- 출력 = "추정 범위 + 가능성", 단정 금지. 급여/비급여 한계를 함수 주석·반환값에 명시.
- 공제 미적용 → 동일 입력=동일 출력(결정론).
