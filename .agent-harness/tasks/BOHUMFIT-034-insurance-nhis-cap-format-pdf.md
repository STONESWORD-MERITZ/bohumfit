# BOHUMFIT-034 실손 계산 UX/산식 보정

## Owner
Codex

## 배경
사용자 확인 결과 독립 실손 계산기에서 다음 문제가 확인됐다.

- `실손 최소공제 자동반영` 카드는 사용자가 입력하거나 검증할 수 없어 화면에 보여줄 필요가 없다.
- 급여 본인부담금/비급여 금액 입력 시 천 단위 콤마가 자동 표시되어야 한다.
- 건강보험 본인부담상한제 초과분은 공단 환급 영역이므로 실손 청구 추정 계산에서 제외하고, 상한까지만 실손 대상 급여 본인부담으로 봐야 한다.
- 배포 화면에서 `리포트 생성 기능을 준비 중입니다`가 계속 보여 `/insurance`가 백엔드 PDF API를 호출하지 않는지 확인해야 한다.

## 작업
- `/insurance` 최소공제 결과 카드를 숨긴다.
- 금액 입력값을 콤마 포함 문자열로 자동 정리한다.
- 소득분위가 선택되어 건강보험 상한제 기준 계산이 가능하면, 실손 청구 가능성/자기부담금 상한제 계산에는 `min(급여 본인부담, 건보 상한)`을 사용한다.
- 청구 가능성 카드에 건보 상한 반영 문구를 표시한다.
- `/insurance`에서 `/api/report/pdf` 호출과 관련 오류 문구가 남아 있지 않은지 확인한다.

## 하지 말 것
- 백엔드 보험 계산 모듈 변경 금지.
- Disclosure 화면 변경 금지.
- 백엔드 PDF 리포트 엔드포인트 변경 금지.
