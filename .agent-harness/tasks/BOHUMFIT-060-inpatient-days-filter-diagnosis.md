# BOHUMFIT-060 입원 행(#47/#48/#49) 집계·필터 변형 진단 (읽기 전용)

## Owner
Cowork(진단·재현, 코드 변경 금지) → Human(수정 태스크 061 승인 결정). 마운트 git 금지·PDF 커밋 금지.

## 증상
샘플 PDF(자동차보험_기본진료정보-장기범.pdf)의 입원 행 #47(0일)·#48(6일)·#49(2일)가
파이프라인을 거치며 "5년 내 입원 이력 2건(리치한방 ~6일·인화재단 2일)"으로 정확히 반영되지 않음.

## 체크리스트
1. 파싱: #47/#48/#49가 basic ftype '입원' 행으로 파싱되는가(상병코드·진료개시일·요양일수·요양기관명 추출).
2. 집계(build_disease_stats): 입원 행이 `inpatient_dates`(set)·`_inpatient_days_map`(date→max)로 어떻게 변형되는가.
3. 필터(_build_q3/q4): `inpatient_count=len(window dates)`·`inpatient_days=sum(max/date)` 산출 결과.

## 가설
- H1 입원 식별이 (disclosure그룹코드, 진료개시일) set 기반 → 같은 코드+날짜 두 입원이 1건으로 병합(과소집계, 일수=max).
- H2 0일 입원(#47)이 별도 날짜면 입원 건수 과집계 + 무의미(당일) 입원이 1건으로 카운트.
- H3 상병코드 공란/`$` 입원 행은 집계 전 `continue`로 drop → 입원 누락.
- H4 진단과='일반의' & 비M54 입원 행 drop(`_keep_basic_general_row`) → 입원 누락.
- H5 병원명이 날짜 키 `setdefault`로 저장 → 같은 날짜 다른 병원(리치한방/인화재단) 표시 손실.

## 제약
- 코드 수정 금지. 원본 PDF(실명·의료정보) 저장소 커밋 금지·마운트 git 금지.
- 실제 PDF는 샌드박스 미보유 → pdf_parser 단계는 정적 분석, 집계·필터 단계는 합성 행으로 실파이프라인 재현.
